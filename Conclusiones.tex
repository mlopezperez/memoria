\chapter{Conclusiones y futuros trabajos}

\section{Conclusiones}

Durante el presente proyecto hemos tenido la oportunidad de exponer los principios de la computación con palabras así como de la valoración de empresas, y hemos unido ambos conceptos, para intentar mejorar el proceso de valoración de una empresa. 

Hemos diseñado e implementado una librería API con la lógica de negocio para valorar empresas, con y sin información cualitativa, así como la implementación de las técnicas de los expertones y el operador LAMA para computación con palabras. 

Por último hemos implementado una aplicación de interfaz de usuario para hacer uso de dicha API. 

\subsection{Conclusiones de investigación}
En el campo de la valoración de empresas utilizando computación con palabras, podemos obtener las siguiente conclusiones:

\begin{itemize}	
	\item El proceso de valoración de una empresa está sujeto a muchas variables díficiles de expresar numéricamente, debido al entorno de incertidumbre en que se produce.
	\item La computación con palabras, basada en la lógica difusa, nos permite añadir información cualitativa, utilizando lenguaje natural, a dicho proceso de valoración.
	\item El modelo de 2-tuplas nos aporta un modelo fácilmente computable para manejar etiquetas lingüísticas sin pérdida de información.
	\item La aportación de información cualitativa al proceso de valoración de empresas puede reducir drásticamente la amplitud del intervalo de posibles valores, facilitando la negociación entre partes.
	\item La manera de agregar dicha información lingüística influye decisivamente en la reducción de dicha amplitud, siendo diferentes las amplitudes obtenidas si agregamos utilizando el operador $LAMA$ a si utilizamos otros métodos como el de los expertones.
	\item En todas las pruebas realizadas, el operador $LAMA$ consigue intervalos más ajustados que el método de los expertones y que la valoración sin información lingüística.
\end{itemize}

\subsection{Conclusiones de formación}
Del proceso de diseño e implementación del presente proyecto hemos adquirido los siguientes conocimientos:

\begin{itemize}
		\item Principios de valoración de empresas. 
		\item Principios de lógica difusa, computación con palabras y representación de información lingüistica utilizando el modelo de dos tuplas.
		\item Agregación de información lingüística utilizando el operador de mayoría $LAMA$.
		\item Agregación de información lingüística utilizando el método de los expertones.
		\item Diseño utilizando Domain Driven Design y arquitectura en N-Capas.
		\item Desarrollo de aplicaciones con tecnología Microsoft .NET, utilizando Windows Presentation Foundation para la capa de presentación y el patrón Model-View-ViewModel.
		\item Desarrollo utilizando la técnica de Test Driven Development.
		\item Realización de tests unitarios utilizando el entorno NUnit.
\end{itemize}
		
\section{Futuros trabajos}
Se proponen las siguientes mejoras a realizar:

\begin{enumerate}
	\item Mejorar la interfaz de usuario para la definición de jerarquías, añadiendo un generador automático de jerarquías y etiquetas, de forma que el usuario pueda abstraerse totalmente de los conceptos de etiqueta lingüística y los valores de estas.
	\item Añadir una visualización gráfica a los resultados de la valoración por los distintos métodos, de forma que el usuario pueda comparar visualmente la mejora en los intervalos cuando se utiliza información lingüística.	
	\item Añadir nuevas metodologías de valoración de empresas, de forma que se pueda comprobar si el ajuste con información lingüística es igualmente efectivo con métodos de valoración menos habituales.	
	\item Encapsular las interfaces principales en una API REST, accesible vía internet usando el protocolo HTTP, de forma que pueda ser desplegada como un microservicio y consumida por cualquier cliente web. 
	\item Una vez definida la interfaz REST, ejecutar un lote de pruebas con distintos casos extremos para comprobar el efecto de la agregación lingüística en dichos casos. Por ejemplo, comprobar la evolución de la amplitud del intervalo de valores conforme aumentamos el número de opiniones, o comprobar el efecto si las opiniones siguen una distribución aleatoria.
\end{enumerate}
